# /set-up-intake
Give a team one fair front door for requests.
## Steps
1. **Design intake** — Define the entry point, fields, and triage using `intake-process` skill.
2. **Clarify rights** — Decide who triages and decides using `raci-matrix` skill.
3. **Connect to goals** — Tie acceptance criteria to outcomes using `okr-alignment` skill.
4. **Record decisions** — Set up where triage calls are logged using `decision-log` skill.
## Output
An intake system: request form, triage cadence, decision rights, and a place decisions are recorded.
Consider following up with `/clarify-roles` to formalise how the team works together.
